# Oracle Review: SEO/AIO/LLMO 100本記事制作計画

実行日: 2026-06-24
Oracle session: `seo-100-article-plan`
Model: GPT-5.5 Pro / browser
元プロンプト: `posts-management/oracle-prompts/seo-100-article-plan-20260624.md`
対象方針: `seo-100-article-direction-20260624.md`

## 結論

Oracleの判断は、現方針の大筋は妥当。ただし、100本を自社HPへ一括投入すると、検索意図の重複、薄い量産感、問い合わせに遠い記事の増加が起きやすい。

採用方針:

- 100本をゼロから生成しない。
- `article_drafts` の approved ドラフトを選別し、公開品質へ変換する。
- 自社HPは問い合わせに近い記事を中心に70本まで。
- note/Qiita/Zenn/GBP/LinkedInは外部導線として30本に分ける。
- P2はGSCの実データを見てから公開判断する。
- 地域名だけ差し替えたページ、市町村ページの量産はしない。

## 採用する優先順位

2. IT担当者がいない会社の外注チェックリスト

6. 金属加工業: 図面確認と見積メモ
7. 放課後等デイサービス: 保護者連絡と出席確認
8. 物流現場: ピッキング確認

### P2: GSCデータを見て判断する30本

P2は、GSCで業種名、課題名、地域名の表示が出た場合にHP詳細化する。表示が出ない場合は、業種別ハブや事例一覧の内部リンク強化に留める。

判断条件:

- GSCで表示が出ている
- 8〜20位に入っている

- 実事例またはapprovedドラフトがある

## バッチ生成方針

本文を100本一括で作らない。10本ずつ、次の順で作る。

1. Batch 1: HP事例記事10本
2. Batch 2: CV直結記事10本
3. Batch 3: 業種別記事10本
4. Batch 4: 課題別記事10本
5. Batch 5: note/Qiita/Zenn外部導線10本

各バッチで必ず出すもの:

- 推奨slug
- title
- meta description
- 狙う検索語
- 関連LP
- 関連する無料資料
- H1
- 本文Markdown
- FAQ 3〜5問
- JSON-LDメモ
- 内部リンク先3つ
- 公開前チェック
- `llms_full.txt` 追記用1行要約

## 公開前チェック

- 1記事1検索意図になっている
- 既存LP、既存記事、事例記事と重複していない
- 関連LP、資料DL、無料相談への導線がある
- 顧客名、売上、成果金額を創作していない
- Before/Afterの数字に根拠がある
- `DX`、`AI`、`システム開発` を抽象語のまま連発していない
- title、description、H1、canonicalが整っている
- FAQが本文に見える形である
- Article/BlogPosting、FAQPage、BreadcrumbListなどのJSON-LDが正しい
- sitemapとllms系へ反映している
- モバイルで横スクロールがない
- CTAクリック、資料DL、問い合わせ導線が計測対象になっている
