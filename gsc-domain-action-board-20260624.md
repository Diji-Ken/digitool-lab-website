### GSC CSV分析手順

GSC画面にログインできるChromeプロファイルでCSVを出力したら、次のように保存して分析する。

```bash
mkdir -p data/gsc
node scripts/analyze-gsc-performance.mjs data/gsc/<gsc-export.csv> --out data/gsc/gsc-performance-report-YYYYMMDD.md
```

`data/gsc/` はGSCの検索クエリやURLを含む運用データ置き場。公開URLでは410にし、FTPデプロイ対象から除外し、gitには `.gitkeep` 以外を入れない。`scripts/audit-private-seo-data.mjs` でこの状態をCI監査する。

このスクリプトで自動抽出するもの:

- ページCSVの場合: ホスト別露出、低CTR URL、8〜20位URL
- クエリCSVの場合: 低CTRクエリ、8〜20位クエリ、重点クラスタ
- 重点クラスタ: さいたま市・地域、DX支援、業務システム、社内ポータル、AI活用・AI研修、MEO・サイテーション、補助金、費用・相場、事例

2026-06-24時点のChrome接続では `sc-domain:digitool-lab.com` が「アクセス権なし」表示だったため、GSC画面操作はログイン済みプロファイル側で実施する。CSVを保存できれば、以後の抽出と改善候補化はローカルで再現できる。
